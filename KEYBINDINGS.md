# ⌨️ Горячие клавиши — шпаргалка для новичка

Это конфигурация на базе **[LazyVim](https://www.lazyvim.org/)**. Здесь собраны самые
нужные клавиши: и стандартные из LazyVim, и кастомные из этого конфига.

---

## 🔑 Главное, что нужно знать

| Понятие | Что это |
|---|---|
| **`<leader>`** | Клавиша-лидер = **`Пробел`**. Большинство команд начинается с него. |
| **`localleader`** | Локальный лидер = **`\`** (обратный слеш). |
| **which-key** | После нажатия `Пробел` подождите ~1 сек — внизу всплывёт **меню** со всеми доступными продолжениями. Это лучший способ всё запомнить: просто жмите `Пробел` и смотрите подсказки. |
| **Модальность** | Vim имеет режимы: `NORMAL` (навигация), `INSERT` (ввод текста, вход по `i`), `VISUAL` (выделение, вход по `v`). `Esc` — всегда возврат в NORMAL. |

> 💡 **Совет новичку:** не пытайтесь выучить всё сразу. Запомните `Пробел` + смотрите
> меню which-key. 90% работы — это файлы (`<leader>ff`), поиск (`<leader>/`),
> переход по коду (`gd`) и git (`<leader>gg`).

## 📁 Файлы и буферы

| Клавиша | Действие |
|---|---|
| `<leader><space>` | Найти файл в проекте (главный поиск) |
| `<leader>ff` | Найти файл (по имени) |
| `<leader>fr` | Недавние файлы |
| `<leader>fn` | Новый файл |
| `<leader>e` | Файловый проводник (дерево слева) |
| `<leader>fe` | Проводник в корне проекта |
| `<leader>bd` | Закрыть текущий буфер (файл) |
| `<leader>bb` | Переключиться на предыдущий буфер |
| `<S-h>` / `<S-l>` | Предыдущий / следующий буфер (Shift+h / Shift+l) |
| `<leader>w` | Сохранить файл |
| `<leader>qq` | Выйти из Neovim |

## 🔍 Поиск

| Клавиша | Действие |
|---|---|
| `<leader>/` | Поиск по тексту во всех файлах проекта (grep) |
| `<leader>sg` | То же — grep по проекту |
| `<leader>sw` | Найти слово под курсором по проекту |
| `<leader>ss` | Поиск по символам в файле (функции, классы) |
| `<leader>sk` | Поиск по горячим клавишам (keymaps) |
| `/` | Поиск в текущем файле (затем `n` / `N` — вперёд/назад) |
| `n` / `N` | Следующий / предыдущий результат **(экран центрируется)** ⭐ |
| `<Esc>` | Убрать подсветку поиска |

> ⭐ В этом конфиге после прыжка по поиску экран автоматически центрируется — удобнее читать.

**Внутри окон поиска (Telescope):**

| Клавиша | Действие |
|---|---|
| `<C-j>` / `<C-k>` | Вниз / вверх по списку результатов ⭐ (кастом) |
| `<CR>` (Enter) | Открыть выбранное |
| `<C-v>` / `<C-x>` | Открыть в вертикальном / горизонтальном сплите |
| `<Esc>` | Закрыть |

## 🧭 Навигация по коду (LSP)

Работает в любом языке с настроенным LSP (Go, Rust, Python, TS, и т.д.).

| Клавиша | Действие |
|---|---|
| `gd` | Перейти к определению |
| `gr` | Показать все использования (references) |
| `gI` | Перейти к реализации |
| `gy` | Перейти к определению типа |
| `gD` | Перейти к объявлению |
| `K` | Показать документацию (hover) под курсором |
| `<leader>ca` | Code Action (быстрые исправления, импорты) |
| `<leader>cr` | Переименовать символ во всём проекте |
| `<leader>cf` | Отформатировать файл вручную ⭐ |
| `]d` / `[d` | Следующая / предыдущая ошибка-диагностика |
| `<leader>cd` | Показать ошибку под курсором |
| `<leader>sd` | Список всех диагностик проекта |

> ⭐ **Важно:** в этом конфиге **автоформат при сохранении отключён**. Форматируйте
> вручную через `<leader>cf`. Также отключён `virtual_text` у ошибок — текст ошибки
> смотрите через `<leader>cd` или наведя курсор.

## ✏️ Полезные базовые приёмы Vim

| Клавиша | Действие |
|---|---|
| `i` / `a` | Вставка перед / после курсора |
| `o` / `O` | Новая строка снизу / сверху + режим вставки |
| `v` / `V` | Выделение символов / строк (Visual) |

| `u` / `<C-r>` | Отменить / повторить |

| `gg` / `G` | В начало / в конец файла |
| `<C-/>` | Открыть терминал |

## 🆘 Если запутались

- **`Esc`** — выйти из любого режима обратно в NORMAL.
- **`Пробел`** и подождать — всплывёт меню с подсказками (which-key).
- **`<leader>sk`** — поиск по всем горячим клавишам.
- **`<leader>L`** — список новостей LazyVim (`:LazyExtras` — управление расширениями).
- **`:checkhealth`** — диагностика проблем конфигурации.
